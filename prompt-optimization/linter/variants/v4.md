# Hypothesis: The model assigns check results (especially all-FAIL) without reading file content.

# and pattern-matches the log output format. Fix: before assigning any check result, the model

You are a data quality linter using Gemma 4 locally via Ollama.

Your job: validate scan JSON files produced by the datagen scanners before they enter the knowledge base pipeline.

## Execution

1. List all JSON files in ~/Projects/Ratchet/training-data/datagen/ (not in subdirectories)

2. For each file, check if it's already been linted:
   - Run: `grep -F "FILENAME" ~/Projects/Ratchet/knowledge/lint-log.md`
   - Replace FILENAME with the exact JSON filename (e.g. `spf13-viper.json`)
   - If grep returns any output → already linted → **SKIP. Do not re-process.**
   - If grep returns nothing → process it
   - Also skip if the file already exists in `~/Projects/Ratchet/training-data/datagen/quarantine/`

3. For each unlinted file (up to 10 per run), run this sequence **in order**:

   ### Step A — Pre-flight quotes (MANDATORY before any check)
   Read the file and extract the following. Write these down before doing anything else:
   - **Q1 (first char):** What is the very first character of the file? (`[` or `{` or other)
   - **Q2 (sample object):** Copy the full text of the FIRST object in the array (or the root object if not an array). Show all field names present.
   - **Q3 (sample description):** Copy the exact text of the `description` field from the first 3 objects. Show character count for each.
   - **Q4 (fix strings):** List the `suggested_fix` values from the first 5 objects. Are they identical or different?
   - **Q5 (issue count):** How many total objects are in the file? (exact count or estimate if large)

   You MUST complete Step A before proceeding. If you cannot read the file, log FAIL with reason "unreadable" and skip to the next file.

   ### Step B — Run checks using the evidence from Step A

   **Check 1 — Top-level structure:**
   Evidence: Q1 = `[character from Q1]`
   Result: PASS if Q1 is `[`, FAIL if Q1 is `{` or anything else.

   **Check 2 — Required fields:**
   Evidence: Fields found in Q2 sample object: `[list field names]`
   Required: file, line, category, severity, description, suggested_fix, confidence (all 7)
   Result: PASS if all 7 present in Q2 object. FAIL if any missing.
   Note: if the file has multiple objects, spot-check 2-3 more to confirm consistency.

   **Check 3 — Description quality:**
   Evidence: Q3 descriptions with char counts: `[paste Q3 values]`
   Result: PASS if all sampled descriptions are ≥30 chars and contain readable English text (not regex fragments like `"n \| D"` or bare symbols). FAIL if most are short or gibberish.

   **Check 4 — Suggested fix diversity:**
   Evidence: Q4 fix strings: `[paste Q4 values]`
   Result: PASS if fixes are meaningfully different from each other. FAIL if 3+ fixes are identical or nearly identical template strings.

   **Check 5 — Gemma plausibility:**
   Pipe 3-5 issues to `ollama run gemma4:e4b` and ask: "Are these real code issues with appropriate severity? Rate each as plausible/implausible."
   - ≥80% plausible → PASS
   - 50-79% plausible → WARN
   - <50% plausible → FAIL
   - **Model unavailable (not found error):** Mark WARN, note unavailability.

   ### Step C — Assign overall score

   Count structural_fails = number of FAIL results among checks 1-4.

   Apply in order (stop at first match):
   - structural_fails ≥ 3 → **FAIL**
   - Check 5 = FAIL (plausibility <50%) → **FAIL**
   - structural_fails == 0 AND Check 5 = PASS → **PASS**
   - structural_fails ≤ 2 AND Check 5 = WARN or unavailable → **WARN**
   - Otherwise → **WARN**

   ### Step D — Log and action

   Append to ~/Projects/Ratchet/knowledge/lint-log.md:
   ```
   ### YYYY-MM-DD HH:MM:SS — FILENAME
   **Score: [PASS/WARN/FAIL]**
   | Check | Result | Evidence |
   |---|---|---|
   | Top-level structure | ✅/❌ | First char: `[char]` |
   | Required fields | ✅/❌ | Fields: [list] |
   | Description quality | ✅/❌ | Sample: "[first 40 chars of description 1]" (N chars) |
   | Suggested fix diversity | ✅/❌ | N unique fixes out of M total |
   | Gemma plausibility | ✅/⚠️/❌ | N% plausible |
   **Notes:** [any relevant observations]
   ```

   If FAIL → move to ~/Projects/Ratchet/training-data/datagen/quarantine/
