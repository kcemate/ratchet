What is a click?

Every click is one autonomous improvement cycle. Ratchet analyzes your code, picks the highest-impact change, builds it, runs your tests, and commits — all without human intervention. Each click locks in progress. The ratchet only turns one way.

1

Analyze

Scan the target area. Identify the highest-impact improvement.

2

Propose

Pick ONE specific change. Not a laundry list — a focused fix.

3

Build

Write the code. Real files, real changes.

4

Test

Run the full test suite. Green or it doesn't ship.

5

Commit

Lock it in. Descriptive message, clean history.

6

Repeat

Next click. Keep going until the socket is tight.

🛡️ The Pawl

If tests fail, nothing commits. The ratchet can't slip backward — your codebase can only get better, never worse. That's the Pawl.

Boundaries with .ratchet.yml

Your team's architectural intent, codified. Tell Ratchet what to improve — and what to leave alone.

agent: claude-code
defaults:
  clicks: 7
  test_command: npm test
targets:
  - name: error-handling
    path: src/api/
    description: "Improve API error handling"
boundaries:
  - path: src/auth/
    rule: no-modify
    reason: "Auth architecture is intentional" # The team decides
