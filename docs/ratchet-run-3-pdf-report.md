## Click 25: The PDF is 3KB. That's suspiciously small.

A4 page with a dark background, multiple sections, category table — and it's 3.3KB? Something's wrong. Either the content is sparse or pdfkit isn't rendering everything.

The likely issue: emojis. PDFKit's built-in Helvetica font doesn't support emoji characters (🧪, ⚠️, 📝, 🔒, ⚡, 📖). They're silently dropped. The category breakdown is missing all its icons.

**Improvement:** Replace emoji with text labels or colored dots. Use amber/green/red circles as category indicators instead of emoji. Or use Unicode symbols that Helvetica actually supports (●, ■, ▲).

**Tests pass?** ✓ — Fixes silent rendering failure.
