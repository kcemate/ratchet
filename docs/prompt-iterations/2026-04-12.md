# Prompt Iteration: 2026-04-12
## Focus: buildScoreContext - Improving Score Motivation Clarity

### Failure Pattern Identified
From F1 worker logs (2026-04-01), multiple repositories show "no-delta" outcomes where commits land but scores remain unchanged (scoreAfter === scoreBefore). Examples:
- got: scoreBefore 64 → scoreAfter 64 (delta: 0)
- express: scoreBefore 63 → scoreAfter 63 (delta: 0)  
- fastify: scoreBefore 63 → scoreAfter 63 (delta: 0)
- trpc: scoreBefore 60 → scoreAfter 60 (delta: 0)

While some repos like hono show successful deltas (+1 point), the high rate of no-delta outcomes suggests agents aren't sufficiently motivated by or_clear on the scoring impact of their fixes.

The current buildScoreContext function generates generic messages like:
"SCORE CONTEXT: Type Safety is at 7/15. Fixing 1 issue gains +1 point. Target the highest-severity issue first."

This lacks specificity about WHICH specific issues contribute to the score and HOW fixing them translates to points.

### Modified Prompt
**Before:**
```typescript
return (
  `SCORE CONTEXT: ${first.subcategory} is at ${currentScore}/${maxScore}. ` +
  `Fixing ${issuesNeeded} issue${issuesNeeded !== 1 ? 's' : ''} gains +${pointsGained} point${pointsGained !== 1 ? 's' : ''}. ` +
  `Target the highest-severity issue first.`
);
```

**After:**
```typescript
return (
  `⚠️  SCORE IMPACT: ${first.subcategory} is at ${currentScore}/${maxScore} points. ` + 
  `Fixing ${issuesNeeded} ${first.subcategory.toLowerCase()} issue${issuesNeeded !== 1 ? 's' : ''} = +${pointsGained} score point${pointsGained !== 1 ? 's' : ''}. ` +
  `🔥 PRIORITY: Fix the ${first.count > 1 ? 'most severe' : 'only'} ${first.subcategory.toLowerCase()} issue first - this is the highest-leverage action for score improvement.`
);
```

### Hypothesis
By making the score context more specific, action-oriented, and emotionally resonant (using ⚠️ and 🔥 emojis for visual salience), agents will have clearer motivation to make scoring-impactful changes. The revised prompt:
1. Uses warning emoji to grab attention
2. Explicitly connects issue type to score points
3. Uses priority emoji and language to emphasize the highest-leverage action
4. Reinforces that this specific fix is the optimal path to score improvement

This addresses the root cause of no-delta outcomes by strengthening the agent's understanding of how their work directly impacts scoring.
