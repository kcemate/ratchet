The `buildIssuePlanPrompt` function (lines 300-376 in `src/core/agents/shell.ts`) suffers from three specific issues:

## Changes Made

### 1. Removed broken escaped output instructions
Removed the two redundant `\\- ANY EXTRA OUTPUT WILL CAUSE ROLLBACK...` lines that had broken escaping.

### 2. Added Pre-Execution Checklist (MUST confirm verbally)
Inserted a 7-item checklist that the agent must verbally confirm BEFORE outputting changes:
1. Acknowledge hard constraints
2. State exact target file
3. Commit to single-file modification
4. State line limit
5. State file limit
6. Commit to smallest possible change
7. Acknowledge rollback on violation

This forces the agent to "own" the constraints before acting.

### 3. Fixed output format to use resolved targetPath
Changed from `MODIFIED: <filepath>` to `MODIFIED: ${targetPath}` so the agent outputs the actual file path, not a literal placeholder.

### 4. Added explicit instruction about no prose
Added: `DO NOT output any prose, analysis, or explanation — ONLY the MODIFIED line and the changes.`
