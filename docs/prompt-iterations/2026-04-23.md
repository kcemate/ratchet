# Prompt Iteration: 2026-04-23
**Target Prompt:** buildAPIBuildPrompt (api agent — free engine)  
**Failure Pattern:** SEARCH/REPLACE format errors + missing verbal commitment + trailing `}` in output

## Current Prompt Analysis

The `buildAPIBuildPrompt` function (lines 226-258 in `src/core/agents/api.ts`) has three specific issues:

1. **No pre-execution checklist** — Unlike `buildIssuePlanPrompt` (fixed 4/22), the API agent receives constraints but never has to actively acknowledge them. The 4/22 fix proved verbal commitment reduces constraint violations.

2. **SEARCH/REPLACE format is under-specified** — The prompt says "exact consecutive lines copied" but doesn't tell the agent HOW to verify correctness. Agents often output SEARCH blocks that don't match due to:
   - Trailing whitespace differences
   - Tab vs space confusion
   - Missing context lines (too short = ambiguous match)
   - Including the `<<<<<<< SEARCH` marker inside the search block

3. **Trailing `}` in output** — F1 logs show `ROLLED BACK (7.0s) — }` as the rollback reason. The `}` is literally the last line of agent output, meaning the agent appended a stray `}` after the MODIFIED line (likely from closing a JSON object or code block in its "thinking").

## Changes Made

### 1. Added Pre-Execution Checklist (same pattern as 4/22 shell fix)
Inserted a 5-item checklist the agent must verbally confirm BEFORE outputting SEARCH/REPLACE blocks:
1. Acknowledge file limit
2. Acknowledge line limit
3. State target file path
4. Commit to exact copy-paste for SEARCH blocks
5. Acknowledge no prose/explanation

### 2. Added SEARCH/REPLACE verification instructions
Added explicit steps for constructing valid SEARCH blocks:
- "Copy EXACTLY 5-10 consecutive lines including the code to change"
- "Do NOT include the `<<<<<<< SEARCH` marker inside your search block"
- "Verify your SEARCH block appears verbatim in FILE CONTENTS above"
- "If the search fails, you used the wrong whitespace — fix and retry"

### 3. Added terminal output guard
Added: `After the last MODIFIED: line, output NOTHING else. No closing braces, no explanations, no JSON.`

This directly targets the trailing `}` pattern.

### 4. Split OUTPUT FORMAT into two sections
Separated the format specification from the critical rules:
- "OUTPUT FORMAT" section: exact syntax
- "SEARCH BLOCK RULES" section: how to construct valid SEARCH blocks
- "TERMINAL RULES" section: what must/must not appear at start/end

## Why This Should Work

- **Active commitment**: Verbal confirmation forces the agent to internalize limits before acting (proven effective in 4/22 shell prompt fix)
- **SEARCH block clarity**: Explicit rules reduce format errors — the `}` rollback suggests agents are outputting malformed blocks or trailing characters
- **Terminal guard**: Directly addresses the trailing `}` by explicitly forbidding anything after MODIFIED:
- **More context**: 5-10 lines (was 3-5) reduces ambiguous matches that cause patch failures
