## Root Cause Analysis
The shell agent's `buildIssuePlanPrompt` function lacked detailed instructions for the SEARCH/REPLACE block format that the torque system requires. While it mentioned "EXACT SEARCH/REPLACE BLOCK format", it only showed a vague placeholder `[agent makes changes...]` instead of the actual required format with markers.

This caused agents to:
1. Output malformed SEARCH blocks (missing markers or incorrect content)
2. Potentially output extra characters after changes (like the "}" character seen in previous iterations)
3. Not follow the exact format torque expects, resulting in warnings (exit code 2) and no score improvements

## Improvement Applied
Enhanced `buildIssuePlanPrompt` in `/Users/giovanni/Projects/ratchet/src/core/agents/shell.ts` with:

### 1. EXACT SEARCH/REPLACE BLOCK Format Specification
```
MODIFIED: ${targetPath}
<<<<<<< SEARCH
[exact consecutive lines copied from the file above — must match exactly, including whitespace]
=======
[replacement lines with fix applied]
>>>>>>> REPLACE
```

### 2. SEARCH BLOCK RULES
- Copy EXACTLY 5-10 consecutive lines from the current file
- Include the code to change PLUS 2-3 lines of context on each side
- Do NOT include the `<<<<<<< SEARCH` marker inside your search block
- Do NOT include the `=======` or `>>>>>>> REPLACE` markers inside either block
- Verify your SEARCH block appears VERBATIM in the current file before outputting
- If unsure, use MORE context lines (8-10) rather than fewer

### 3. OUTPUT FORMAT (showing exactly what to output)
```
MODIFIED: ${targetPath}
<<<<<<< SEARCH
[exact consecutive lines copied from the file above — 5-10 lines minimum]
=======
[replacement lines]
>>>>>>> REPLACE
```

### 4. TERMINAL RULES (Directly addresses extra character output)
- Do NOT output anything before the first MODIFIED: line
- Do NOT output anything after the last REPLACE: line
- After the last REPLACE: line, output NOTHING else — no closing braces, no explanations, no JSON, no backticks
- Do NOT reformat, restyle, or change whitespace outside the fix

## Expected Outcome
This change should eliminate torque warnings (exit code 2) and improve score rates by:
1. Requiring agents to use the exact SEARCH/REPLACE format expected by torque
2. Providing clear rules to prevent malformed SEARCH blocks
3. Explicitly forbidding any output after the REPLACE: line (including stray "}" characters)
4. Giving agents a concrete example of the exact output format expected
