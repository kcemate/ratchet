# Prompt Iteration: 2026-04-24

## Agent: Shell Agent (buildIssuePlanPrompt)

## Failure Pattern Addressed
Lone "}" character in agent output causing torque rollbacks with reason "}"

**Evidence from F1 Worker Logs (April 1, 2026):**
- hono-torque.log: `{"reason":"}","msg":"[ratchet] click 1 ROLLED BACK (9.2s) — }"}`
- express-torque.log: `{"reason":"}","msg":"[ratchet] click 1 ROLLED BACK (7.0s) — }"}`

## Root Cause Analysis
The shell agent's `buildIssuePlanPrompt` function was not specific enough about the required output format for code changes. While it instructed agents to output changes, it did not:
1. Specify the exact SEARCH/REPLACE block format required by the torque system
2. Provide detailed rules for constructing valid SEARCH blocks
3. Explicitly forbid trailing characters after changes (particularly the "}" character)

## Improvement Applied
Enhanced `buildIssuePlanPrompt` in `/Users/giovanni/Projects/ratchet/src/core/agents/shell.ts` with:

### 1. EXACT SEARCH/REPLACE BLOCK Format Specification
```
MODIFIED: ${targetPath}
<<<<<<< SEARCH
[exact consecutive lines copied from current file — must match exactly, including whitespace]
=======
[modified lines with fix applied]
>>>>>>> REPLACE
```

### 2. SEARCH BLOCK RULES
- Copy EXACTLY 5-10 consecutive lines from the current file
- Include the code to change PLUS 2-3 lines of context on each side
- Do NOT include the `<<<<<<< SEARCH` marker inside your search block
- Do NOT include the `=======` or `>>>>>>> REPLACE` markers inside either block
- Verify your SEARCH block appears VERBATIM in the current file before outputting
- If unsure, use MORE context lines (8-10) rather than fewer

### 3. TERMINAL RULES (Directly addresses the "}" failure)
- Do NOT output anything before the first MODIFIED: line
- Do NOT output anything after the last REPLACE: line
- **After the last REPLACE: line, output NOTHING else — no closing braces, no explanations, no JSON, no backticks**
- Do NOT reformat, restyle, or change whitespace outside the fix

## Expected Outcome
This change should eliminate the "}" rollback failures by:
1. Requiring agents to use the exact SEARCH/REPLACE format expected by torque
2. Providing clear rules to prevent malformed SEARCH blocks
3. Explicitly forbidding any output after the REPLACE: line (including stray "}" characters)
