# Prompt Iteration: 2026-04-05
## Focus: buildIssuePlanPrompt - Preventing File Path Hallucination

### Failure Pattern Identified
From F1 worker logs (2026-04-01), the fastify repo experienced a critical failure:
- Agent attempted to edit `lib/fastify.js` which does not exist in the repository
- Search text was not found in `lib/hooks.js`, causing the click to be skipped
- The click rolled back with "build failed"

This is a clear case of **file path hallucination** — the agent invented file paths instead of working only on the provided target path. The API agent prompt already includes "Do NOT invent or guess file paths" but the message was ignored. The issue appears to be that the warning is buried in the STRICT RULES section rather than being prominently featured.

### Modified Prompt
**Before:**
```ts
return (
  `⚠️  CRITICAL: Do NOT refactor, restructure, or rewrite functions\n` +
  `⚠️  CRITICAL: Do NOT rename variables, extract helpers, or "improve" unrelated code\n` +
  `⚠️  CRITICAL: Do NOT add new dependencies or change public function signatures\n` +
  `⚠️  CRITICAL: Do NOT change formatting, whitespace, or style in untouched lines\n\n` +
  `You are a code improvement assistant. Fix the top issue in ${targetPath}.\n\n` +
  ...
);
```

**After:**
```ts
return (
  `⚠️  CRITICAL: Do NOT invent, guess, or hallucinate file paths. Work ONLY on the provided target path.\n` +
  `⚠️  CRITICAL: Do NOT refactor, restructure, or rewrite functions\n` +
  `⚠️  CRITICAL: Do NOT rename variables, extract helpers, or "improve" unrelated code\n` +
  `⚠️  CRITICAL: Do NOT add new dependencies or change public function signatures\n` +
  `⚠️  CRITICAL: Do NOT change formatting, whitespace, or style in untouched lines\n\n` +
  `You are a code improvement assistant. Fix the top issue in ${targetPath}.\n\n` +
  ...
);
```

### Hypothesis
By adding a specific, prominently marked (⚠️) warning about file path hallucination at the very beginning of the prompt, the agent will be more likely to adhere to the constraint and avoid inventing file paths. The visual marker and placement increase cognitive salience.

### Testing Plan
1. Apply the change to `src/core/agents/shell.ts`
2. Run `npm test` to ensure no regressions
3. Run a local torque test on a sample repo to validate behavior
4. Monitor F1 worker logs for reduction in "search text not found" errors
